1. Escolher um banco de dados relacional, criar e definir como será o acesso ao banco de dados.
Pode utilizar ORM se desejar.
a. Pode escolher um framework apresentado em sala de aula, um framework qualquer, ou
implementar o acesso e manipulação direta os dados do banco.
2. Realizar a pesquisa e definição de um framework para conversão de objetos para JSON.
3. O aluno irá gerar um vetor ordenado (com números inteiros) e deve randomizar o vetor.
a. O algoritmo precisa ramdomizar o vetor (embaralhar de forma aleatória as posições). A
cada execução o algoritmo precisa embaralhar o vetor em uma ordem diferente.
b. O vetor deve conter 50 mil números não duplicados (ex: de 1 até 50.000).
c. Salvar o vetor randomizado em um banco de dados relacional (de sua escolha), contendo:
uma tabela com o nome do vetor sua descrição, e outra para salvar a ordem aleatória.
d. Apresentar o resultado em Json na tela do navegador ou postman.
e. Realizar a tomada de tempo de 3 execuções.
4. Realizar a pesquisa e definição de um algoritmo para ordenação de um vetor de números inteiros.
a. Ler o vetor no banco de dados, organizar ele, e apresentar em tela.
b. Realizar a tomada de tempo de 3 execuções.
c. Apresentar o resultado em Json na tela do navegador ou postman.
5. Escolher uma ou mais Librarys para implementar gráficos e baixar uma imagem para o
computador. Deve-se gerar os seguintes gráficos:
a. Scatter Plots – Gráfico de Dispersão
b. Line Charts – Gráfico de linhas
c. Bar charts – Gráfico de Barras
d. Bubble Charts – Gráfico de Bolhas
e. Dot Plots – Gráfico de pontos
6. Realizar a elaboração de um diagrama com imagens apresentando o flow (fluxo) do seu projeto.
Considere a entrada de dados via HTML e a saída de dados utilizando HTML e JSON.
